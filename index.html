About this website:

This website is a prototype, and it's still WIP.

If it ever gets developed into a real thing it will be a place to host coding challenges for programmers.

Unlike most corporate hackatons, this is not a business case. It's actually about programming.

Unlike traditional hackatons this website relies on an homemade engine that supports a "game-like" graphical interface, in which the effects of your code can be seen in real-time.

The rules are simple:
- The language is Javascript. You are given the update() function to fill in.
- update() has one parameter, the state: an object containing everything you need to know about the simulated environment and all the actions that can be performed to affect it.
- update() will be called three times a second.

Challenge 0: Tutorial

This is mostly to get everyone familiarized with the system. The simulated "world" is a 10x10 grid, with the origin on the bottom-left. The state contains two objects:
- target, with an attribute position
- player, also with a position, but also exposing a move() function.
All positions, coordinate, movements, etc.. are expressed as length 2 arrays.
