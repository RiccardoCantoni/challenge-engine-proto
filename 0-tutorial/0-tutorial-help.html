Challenge 0: Tutorial

This is mostly to get everyone familiarized with the system. The simulated "world" is a 10x10 grid, with the origin on the bottom-left. The state contains two objects:
- target, with an attribute position
- player, also with a position, but also exposing a move() function.
All positions, coordinate, movements, etc.. are expressed as length 2 arrays.

Get player to the target.

- You can only move up, down, right, or left, one cell at a time.
This means that the only allowed movements are [0,1], [0,-1]...
Attempting to move by an invalid vector will stop the simulation.
- You can only move once per update(). Attempting to move more than once will stop the simulation.
- The challenge is completed when player is in the same position as target.

Some general considerations:
- The console works normally
- While the challenges definitely don't need any external code, importing 3rd-party libraries is not suggested, as it will probably break everything.
- Yes, this website uses eval() to read your code. This means that you can break it if you really try. It doesn't really cause any harm since there is no backend, so have fun.
