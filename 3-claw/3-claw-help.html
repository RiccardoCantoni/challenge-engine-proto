Challenge 3: Claw Machine

Change of scenery: this time you will be controlling a mechanical claw.
Not only it moves, but it can also pick-up, transport and release coloured blocks.

Re-arrange the blocks until they are in the same order as the reference stack.

The claw has a move() method which works left-to-right and a toggle() method which picks up or releases a block.

Blocks can only be placed within the allowed 3x6 area (it has a thin grey outline).

The state also contains two arrays: blocks and target, that hold the movable blocks in the main game area, and the reference blocks, respectively.

- You can move the claw, left or right, one cell at a time.
- You can only move once per update().
- You can only toggle once per update().
- You cannot move and toggle in the same update().
- The challenge is completed when blocks represents the same color arrangement as target.
- state.U is an object exposing a number of utility methods for doing mathematical operations on vectors. You won't necessarily need to use any of them, but a brief documentation can be accessed from the challenge page though the Utils link.
